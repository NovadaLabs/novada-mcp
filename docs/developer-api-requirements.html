1 背景与目标

Ethan 提出方向:把 Novada developer-api 包装成 MCP(Model Context Protocol)工具,让客户的 AI agent(Claude Desktop / Cursor / Cline / Windsurf 等)可以直接通过自然语言调用 Novada 的能力 — 包括查用量、管账户、续套餐、监控抓取任务,而不需要客户手动登 dashboard。

战略价值 — 这是差异化 USP
BrightData / Tavily / Firecrawl 等竞品都做了 hosted MCP,但它们的 MCP 工具只覆盖数据获取(search / scrape),不覆盖账户管理。Novada 如果做出"AI agent 自己管你的 Novada 账户",将是市场上第一家。客户体验从「我得登 dashboard 查余额」变成「我问 AI 余额还有多少 + 帮我续费」。

我们(Novada Labs / Tong Wu)负责的:把所有可调 endpoint 包装成 MCP tool,处理 auth / 参数验证 / 错误码 / 用户确认逻辑。后端(甫栋 / domestic team)负责的:给出完整文档 + 补几个目前缺的 endpoint。
