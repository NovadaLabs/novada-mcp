// V4 and V5 MUST be sequential — same URL, same session

t2 = NOW()
v5_result = CALL(novada_extract, { url: "https://news.ycombinator.com" })

### Execution SOP

```pseudocode
ROUNDS = 50
WARMUP = 1  // 1 warmup round per (competitor × task), not counted

FOR EACH competitor IN [novada, firecrawl, tavily, oxylabs]:
  FOR EACH task IN supported_tasks(competitor):
    results = []
    
    // Warmup (discarded)
    CALL(competitor, task)
    SLEEP(1000)
    
    FOR round = 1 TO ROUNDS:
      t = NOW()
      TRY:
        response = CALL(competitor, task)
        latency = NOW() - t
        quality = scoreContent(task.id, response.text)
        af = scoreAgentFriendliness(competitor, response, response.text, false)
        
        // NEW in 0.7.8: check for new features in Novada responses
        features = {}
        IF competitor == "novada":
          features.has_chainable = response.text.includes("## Chainable Output")
          features.has_remember = response.text.includes("remember:")
          features.has_source = response.text.includes("source: live") OR response.text.includes("source:")
          features.has_agent_instruction_success = response.text.includes("agent_instruction:")
        
        results.PUSH({ round, status: "ok", latency, quality, af, features })
      CATCH error:
        latency = NOW() - t
        results.PUSH({ round, status: "error", latency, error: error.message })
      
      // Rate limit protection
      IF competitor == "firecrawl": SLEEP(2000)  // 429-prone
      ELSE: SLEEP(500)
    
    SAVE(competitor, task.id, results)

GENERATE_REPORT(allResults)
```

### Scoring Criteria

**Content Quality (1-5):**
- Q5 = target content fully present, agent-ready
- Q4 = most content present, minor gaps
- Q3 = partial content
- Q2 = minimal relevant content
- Q1 = content absent or irrelevant

Use the `scoreContent()` function from `run.mjs` — it has per-task scoring heuristics.

**Agent-Friendliness (1-5):**
5 criteria, 1 point each:
1. `has_agent_instruction` — explicit next-step guidance in success response
2. `error_is_structured` — errors are JSON/structured, not raw stack traces
3. `has_status_field` — response includes status/ok/error indicator
4. `output_is_chainable` — URLs in structured format (not just embedded in text)
5. `low_boilerplate` — response is mostly useful content, not wrapper HTML

Use `scoreAgentFriendliness()` from `run.mjs`.

**NEW scoring for 0.7.8:** For Novada responses, also track:
- `chainable_output_present` (boolean) — `## Chainable Output` section exists
- `remember_present` (boolean) — `remember:` line exists
- `source_present` (boolean) — `source: live|wayback|cache` exists
- `json_mode_valid` (boolean, T3/T4 only) — JSON format produces valid JSON
