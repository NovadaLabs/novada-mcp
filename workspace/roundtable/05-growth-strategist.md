# Growth Strategist

## 1. Fastest Path to 1,000 npm Downloads/Month

**Intercept frustrated users where they already are:**

- **Show HN post** targeting "open-source web scraping MCP" -- HN drives 200-500 installs in 48 hours if it lands mid-page. Time it for a Tuesday 9am ET.
- **Reddit r/codex and r/ClaudeAI** -- Firecrawl credit complaints are live demand. Post a comparison (not attack) showing Novada's flat pricing. One genuine "I switched" post outweighs ten ads.
- **MCP directory listings** -- Smithery, Glama, mcp.so. These are where agents discover tools. Free, high-intent traffic.
- **Claude community integrations** -- Get listed in Claude Desktop's recommended servers. One integration partnership = 500+ sustained monthly installs.
- **GitHub Awesome lists** -- awesome-mcp-servers, awesome-web-scraping. PRs take 10 minutes.

Secondary: dev blog posts solving specific pain points (e.g., "Scraping JS-heavy sites without Puppeteer") that rank for long-tail keywords.

## 2. This Week: One Action

**Post on Show HN.** Write a concise launch post: "Novada MCP -- 35+ tools for web scraping/search at $1/1k pages." Include a 30-second demo GIF. Nothing else matters until you have public signal.

## 3. Common Mistake at This Stage

Building more features instead of distributing what exists. You have 35+ tools, 14 chapters of docs, and a hosted endpoint. The product is not the bottleneck -- obscurity is. Every hour spent adding tool #36 instead of getting tool #1 in front of users is wasted. Ship the Show HN before writing another line of code.
